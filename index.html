A Context-Free Grammar (CFG) is a formal system used in computer science to describe the structure of languages. It is widely used in compiler design, programming languages, and syntax analysis. A CFG consists of a set of production rules that define how strings can be generated from a start symbol. These rules are called "context-free" because they can be applied regardless of the surrounding symbols.

A CFG is defined by four components: a set of non-terminals, a set of terminals, a set of production rules, and a start symbol. Non-terminals are variables like S, A, and B that can be expanded further, while terminals are the actual symbols of the language such as a, b, or id. Production rules describe how non-terminals can be replaced with combinations of terminals and non-terminals.

CFGs are powerful enough to represent most programming language constructs such as expressions, loops, and conditional statements. They are used in the design of parsers and compilers to check whether a program follows the correct syntax. However, CFGs cannot represent all types of languages, especially those requiring context sensitivity.

In practical applications, CFGs are used along with parsing techniques to analyze input strings and generate parse trees. These trees help in understanding the hierarchical structure of the language and are essential for further stages like semantic analysis and code generation.

A parse tree is a graphical representation that shows how a string is derived from a context-free grammar. It represents the hierarchical structure of the derivation process. The root of the tree is the start symbol, and each internal node represents a non-terminal, while the leaf nodes represent terminals.

In a parse tree, each production rule is applied by expanding a non-terminal node into its children. The children correspond to the symbols on the right-hand side of the production rule. By following the tree from root to leaves, we can reconstruct the generated string.

Parse trees are important in compiler design because they help in syntax analysis. They allow the compiler to verify whether a given string belongs to the language defined by the grammar. If a valid parse tree can be constructed, the string is accepted; otherwise, it is rejected.

Parse trees also play a crucial role in further stages of compilation such as semantic analysis and code generation. They help in understanding operator precedence, associativity, and the overall structure of expressions in programming languages.

  • Derivation is the process of generating a string from the start symbol using production rules.
  • Each step replaces a non-terminal with the right-hand side of a production rule.
  • Leftmost derivation expands the leftmost non-terminal first at every step.
  • Rightmost derivation expands the rightmost non-terminal first.
  • Both derivations generate the same language but may have different intermediate steps.
  • Derivations are used to construct parse trees and verify string validity.

What is a Context-Free Grammar?

A Context-Free Grammar (CFG) is a set of rules that define how to generate strings in a language. It consists of:

  • Non-terminals: Variables (usually uppercase letters like S, A, B)
  • Terminals: Actual symbols of the language (lowercase letters, numbers, etc.)
  • Production rules: Rules showing how non-terminals can be replaced
  • Start symbol: The initial non-terminal (usually S)

Example: S → aSb | ε

This grammar generates strings like: ε, ab, aabb, aaabbb, etc.

Reading a parse tree:

  • Start at the root (usually S)
  • Each internal node is a non-terminal
  • Each leaf node is a terminal
  • Following the leaves left-to-right gives you the string

Leftmost vs Rightmost Derivation

When deriving a string, we need to choose which non-terminal to expand first. There are two common strategies:

Leftmost Derivation

Always expand the leftmost non-terminal first.

S → aSb

aSb → aaSbb

aaSbb → aabbb

Rightmost Derivation

Always expand the rightmost non-terminal first.

S → aSb

aSb → aSbb

aSbb → aaSbbb

aaSbbb → aaabbb

Both strategies can generate the same string but may produce different intermediate steps.

5

Grammar Ambiguity

A grammar is ambiguous if a string can be parsed in multiple different ways (has multiple parse trees).

Ambiguous Grammar: E → E+E | E*E | a

The expression "a+a*a" can be parsed as:

  • (a+a)*a
  • a+(a*a)

These have different meanings in arithmetic!
